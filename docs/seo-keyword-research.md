# PASO 0 — Investigación SEO del nicho keto en español

> Base para construir las 30 entradas de blog (y reforzar recetas/landing). **1 keyword = 1 página**, sin canibalización. Cada blog se redacta a mano, con fuente médica verificada por WebFetch.

## 1. Panorama de intención de búsqueda

El nicho keto en español mezcla 4 intenciones:
- **Informacional / principiante:** "qué es la dieta keto", "keto para principiantes", "qué es la cetosis" → quieren entender y empezar.
- **Cálculo / acción:** "calorías en keto", "cómo calcular macros", "cuántos carbohidratos al día" → quieren números y un método.
- **Receta / comida:** "pan keto sin harina", "postres keto sin azúcar", "snacks keto" → intención transaccional-blanda (quieren hacerlo).

> **REGLA DURA DE CONTENIDO (orden de Ferney):** SOLO estilo de vida, comida y prevención. **NO se tocan temas de salud delicada / enfermedades** (diabetes, hipertensión, hígado graso, tiroides, colesterol y similares). Esos temas YMYL quedan FUERA del blog.

Falta contenido que combine **evidencia médica real citada + tips accionables + conexión con un método/producto práctico**, con **E-E-A-T real** (Ferney como autor + canal de YouTube con recetas propias = "experiencia" demostrable, la primera E de E-E-A-T que casi nadie tiene en este nicho). Ahí ganamos long-tails específicos.

## 3. Por qué rankean los primeros → técnicas a replicar (adaptadas, sin copiar)

1. **Intención cubierta al 100%** y formato largo bien estructurado (H1 único + H2/H3 jerárquicos).
2. **Schema correcto** por tipo (Recipe / Article / FAQ / Breadcrumb / ItemList).
3. **E-E-A-T:** autor identificado con bio (Ferney), fuentes reales enlazadas, "experiencia" (YouTube). Clave en YMYL (salud).
4. **Clusters internos:** se enlazan entre temas relacionados y a la página que convierte (libro).
5. **Mobile-first + velocidad** (88% del tráfico es móvil).
6. **FAQ** al final para capturar preguntas long-tail (y FAQPage schema).

## 4. Reglas aplicadas a cada página (resumen)
- Slug = keyword exacta, 3-5 palabras, minúsculas, guiones, **sin acentos ni ñ** (n), sin relleno (el/la/de/para).
- Slug + Title + H1 + keyword del 1er párrafo → **alineados a la misma query real**. Title 50-60 car con keyword al inicio. Meta description 150-160 con keyword + gancho de CTR.
- Article + BreadcrumbList (+ FAQPage cuando aplique) schema. Autor = Ferney (Person) con bio.
- Enlace interno: al libro relevante (anchor con keyword) + 2-3 a blogs del mismo cluster + de vuelta al hub.
- Imagen hero on-domain, webp, nombre con keyword, alt descriptivo, lazy.
- AdSense ≤30%, distribuido (tras intro, in-content cada 4-5 párrafos, 1 al final), a ≥150px de CTAs.
